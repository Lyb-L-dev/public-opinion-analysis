# 社会舆情分析系统 - 系统说明文档

---

## 第一章 系统概述

### 1.1 系统定位

社会舆情分析系统是一款基于微博热搜数据的舆情监测与分析平台。系统自动采集微博热搜榜单、相关文章与用户评论，通过自然语言处理（NLP）和机器学习技术进行情感分析、关键词提取、异常检测和影响力评估，以可视化仪表盘的形式呈现分析结果，帮助用户及时掌握舆情动态。

### 1.2 核心技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| Web 框架 | Python Flask | 3.1.2 |
| 数据库 | MySQL | - |
| 前端 UI | Bootstrap 5 | 5.3.0 |
| 图标库 | Font Awesome | 6.4.0 |
| 图表库（趋势/面积/柱状/饼图） | ApexCharts | 3.45.0 |
| 图表库（地图/词云/散点/力导向图） | ECharts | 5.4.3/5.5.0 |
| 实时推送 | WebSocket | - |
| 中文分词 | jieba | - |
| AI 报告生成 | MiniMax API | - |
| 异常检测 | 孤立森林 (Isolation Forest) | - |

### 1.3 系统架构

```
数据采集(crawlers) --> 数据处理(NLP/ML) --> 数据存储(MySQL) --> 分析服务(services) --> 前端展示(templates)
                                                                                    |
                                                             实时推送(WebSocket) -----+
```

## 第二章 用户认证模块

### 2.1 登录

**页面**：`/auth/login`

**功能描述**：
- 登录页面采用科技感粒子网络动画背景（Canvas 绘制 80 个粒子，距离小于 120px 时连线）
- 左侧品牌展示区显示系统名称"社会舆情分析系统"及描述文字
- 显示三个数据指标卡片（今日采集数据 / 风险预警 / 爆发词），使用毛玻璃透明效果
- 右侧登录面板包含用户名和密码输入框
- 支持"记住我"功能
- 密码输入框支持显示/隐藏切换（眼睛图标）
- 登录失败时显示红色横幅错误提示（带抖动动画）
- 提供"忘记密码"和"立即注册"链接入口

**数据接口**：`POST /auth/login`（传统表单提交）

### 2.2 注册

**页面**：`/auth/register`

**功能描述**：
- 与登录页采用相同的科技风粒子动画背景
- 注册表单包含：用户名、密码、确认密码
- 提供密保问题下拉选择框（从预设安全问题列表中选取）
- 密保答案输入框
- 注册成功后显示提示信息并提供"立即登录"链接
- 注册失败显示错误信息

**数据接口**：`POST /auth/register`（传统表单提交）

### 2.3 忘记密码

**页面**：`/auth/forgot_password_step1` → `/auth/forgot_password_step2` → `reset_password_success`

**功能描述**：
- **第一步**：输入用户名进行身份验证，系统根据用户名查找对应的密保问题
- **第二步**：回答密保问题，并设置新密码（含确认密码）
- **成功页面**：显示"密码重置成功"提示，引导用户跳转到登录页

**数据接口**：
- `POST /auth/forgot_password_step1`
- `POST /auth/forgot_password_step2`

### 2.4 退出登录

**功能描述**：位于用户下拉菜单中，点击后清除会话并重定向至登录页面。

**数据接口**：`GET /auth/logout`

## 第三章 主仪表盘

**页面**：`/dashboard/`

### 3.1 实时微博热搜

**功能描述**：
- 通过 WebSocket 连接到服务器实时接收微博热搜数据
- 以表格形式展示热搜排名、热搜标题（可点击跳转至微博搜索）、热度值（千位分隔符格式化）
- 显示数据最后更新时间
- "刷新热搜"按钮可手动重连 WebSocket
- WebSocket 断开后自动每 5 秒尝试重连

**数据接口**：WebSocket `ws://localhost:8765`

### 3.2 数据统计卡片

**功能描述**：
- 展示 4 个关键指标卡片，带有数字递增滚动动画：
  - 热点事件总数
  - 评论总数（含进度条）
  - 平均点赞数
  - 平均情感分百分比
- 数据通过后端渲染传入模板

### 3.3 情感趋势分析图

**功能描述**：
- ApexCharts 面积图，展示情感评分随时间的变化趋势
- 支持三个时间范围切换按钮：本周 / 本月 / 本季
- X 轴日期标签旋转 -90 度以优化显示
- 点击按钮触发 AJAX 请求切换数据

**数据接口**：`GET /dashboard/get_sentiment_trend?time_range=week|month|quarter`

### 3.4 情感分布环形图

**功能描述**：
- ApexCharts 环形图（donut），展示全部事件的正面/中性/负面情感比例
- 三色分区，清晰展示整体情感倾向

### 3.5 热门事件卡片网格

**功能描述**：
- 以卡片网格形式展示各热门事件
- 每张卡片包含：
  - 事件标题
  - 情感标签（正面/负面/中性，不同颜色）
  - 风险等级标签（高/中/低）
  - 热度值
  - 抓取时间
  - 评论数、总点赞数、情感分统计
  - 情感分布三色进度条（正面绿/中性蓝/负面红）
- 操作按钮：
  - **查看详情**：跳转至事件详情页
  - **深度分析**：跳转至事件深度分析页
  - **收藏切换**：实心/空心星标切换（POST AJAX 请求）
- "刷新数据"按钮（含加载旋转动画），支持 `Ctrl+R` 键盘快捷键
- 支持事件监控添加

**数据接口**：
- `POST /dashboard/refresh`
- `POST /favorite/toggle/<event_id>`

### 4.4 热度与情感关联散点图

**功能描述**：

### 4.6 事件热度趋势图

**功能描述**：

- 左轴：总热度值
- 右轴：平均热度值

**数据接口**：`GET /visualization/api/event_heat_trend?range=week|month`

### 4.7 评论点赞分布柱状图

**功能描述**：

## 第五章 热点事件管理

### 5.1 事件列表页

**页面**：`/dashboard/hot-event-list`

**功能描述**：
- 以表格形式展示所有热点事件，包含 7 列：
  - 事件标题（长标题截断显示）
  - 抓取时间
  - 热度值
  - 情感评分
  - 评论数
  - 风险等级标签（高=红色 / 中=橙色 / 低=绿色）
  - 操作按钮（查看详情 / 深度分析）
- **搜索功能**：输入关键词实时筛选事件标题（客户端 JS 实现）
- **风险等级筛选**：下拉框可按高风险/中风险/低风险/全部筛选
- 搜索和筛选联动
- **分页导航**：首页/上一页/页码/下一页/尾页，支持输入页码跳转

**数据接口**：服务端渲染分页，`GET /dashboard/hot-event-list?page=N`

### 5.2 事件详情页

**页面**：`/dashboard/event/detail/<event_id>`

**功能描述**：
- **事件信息卡片**：
  - 事件标题
  - 元数据网格：抓取时间、事件热度、情感评分、风险等级（动态颜色）、总评论数
  - 情感分布三色进度条（正面/中性/负面百分比）
- **文章详情卡片**：
  - 作者名、发布时间
  - 文章全文内容
  - 互动数据：点赞数、评论数、转发数
  - 情感评分
- **评论列表**：
  - 评论统计标签：正面/中性/负面数量
  - "重新爬取评论"按钮（带加载动画和禁用状态，爬取完成后自动刷新页面）
  - 每条评论展示：用户名、发布时间、情感标签、内容、点赞数

**数据接口**：
- `POST /dashboard/api/crawl-comments/<event_id>`（触发后台爬取评论）

### 5.3 事件深度分析页

**页面**：`/dashboard/event/deep_analysis/<event_id>`

**功能描述**：
- **事件基础信息卡片**：事件标题、抓取时间、热度、整体情感分、风险等级、评论总数、总点赞数
- **四个情感统计卡片**：
  - 正面评论数 + 占比百分比 + 进度条
  - 中性评论数 + 占比百分比 + 进度条
  - 负面评论数 + 占比百分比 + 进度条
  - 评论平均情感分 + 趋势方向（上升/下降/平稳箭头）
- **智能洞察卡片**（有条件显示）：
  - 风险等级徽章
  - 关键发现列表（带圆点标记）
  - 风险评估：趋势方向、趋势分析、风险理由
  - ML 模型分析：异常状态、异常严重度、影响力评分
- **相关文章列表**：
  - 文章情感统计（正面/中性/负面数量）
  - 文章逐条展示：作者、时间、内容摘要（截断 300 字符）、点赞/评论/转发数、情感标签
- **评论情感趋势变化图**（ApexCharts 面积图）：
  - 支持按小时 / 按日期两种粒度切换
- **评论情感分布环形图**（ApexCharts donut，渐变填充 + 展开动画）
- **评论发布时间分布柱状图**（ApexCharts）
- **评论热点关键词云**（后端生成 Base64 图片）：
  - "刷新"按钮可重新请求后端生成新词云
- **评论明细列表**：
  - 情感筛选按钮（全部 / 正面 / 负面）
  - 每条评论：用户名、时间、内容、点赞数、情感分
  - 分页控件（上一页/下一页 + 页码信息）

**数据接口**：
- `GET /dashboard/event/deep_analysis/<event_id>/get_trend?range=hour|day`
- `GET /dashboard/event/deep_analysis/<event_id>/refresh_keywords`

**功能描述**：

**功能描述**：
- 大/中/小三种尺寸的标签云展示热门关键词

## 第七章 AI 智能分析模块

**页面**：`/enhanced/ml_dashboard`

### 7.1 全局舆情状态

**功能描述**：
- 顶部彩色横幅展示四组核心指标：
  - **舆情指数**：0-100 分，基于整体平均情感分计算
  - **风险等级**：动态颜色徽章（红色=高风险 / 橙色=中风险 / 绿色=低风险）
  - **舆情趋势**：上升/下降/平稳（含箭头图标和颜色区分）
  - **情感分布**：三色条形图 + 百分比图例（正面/中性/负面）
- 显示数据最后更新时间
- 提供刷新按钮（带自旋动画）

**数据接口**：`GET /enhanced/api/ml/comprehensive-report?days=30`

### 7.2 异常事件检测

**功能描述**：
- 基于**孤立森林算法**（Isolation Forest）的异常检测
- 展示异常统计：异常总数、高风险/中风险/低风险数量
- 异常事件卡片列表（最多 6 张），每张包含：
  - 事件标题
  - 异常分数
  - 异常类型标签（热度异常 / 评论激增 / 情感骤降 / 情感激增 / 互动激增）
  - 异常原因说明
  - 左侧色条指示严重度

**数据接口**：`GET /enhanced/api/ml/anomaly-detection?days=7`

### 7.3 事件影响力评分

**功能描述**：
- 展示事件影响力平均分和最高分统计
- 影响力排名列表（Top 8），每项包含：
  - 排名徽章（前 3 名金/银/铜渐变样式）
  - 事件标题
  - 影响力分数进度条
  - 展开按钮：点击后可查看五因子分解详情：
    - 热度因子
    - 评论文本量因子
    - 情感影响因子
    - 互动率因子
    - 传播范围因子
  - 每个因子显示标签、数值和进度条

**数据接口**：`GET /enhanced/api/ml/influence-scoring?limit=8`

### 7.4 AI 智能分析报告

**功能描述**：
- 基于 **MiniMax 大模型**生成的智能分析报告
- AI 生成标识徽章（绿色=大模型生成 / 橙色=规则生成）
- 报告内容包含：
  - **关键发现**：带灯泡图标和左边框强调的多条核心发现
  - **风险评估**：风险等级 + 风险理由
  - **舆情态势总结**：整体态势描述
  - **应对建议**：3 列网格卡片布局，每张含优先级标签

#### `utils/text_utils.py` - 文本处理工具

**定位**：NLP 文本处理核心模块，提供情感分析、关键词提取和事件分类功能。

**一、情感分析**：
- `analyze_sentiment(text)` - 使用 **SnowNLP** 计算基础情感分（0-1），再通过 **自定义情感词典**（150+ 正面词、150+ 负面词）进行校准加权
- `batch_analyze_sentiment(texts)` - 单进程批量分析
- `batch_analyze_sentiment_parallel(texts)` - 多进程并行批量分析
- `get_sentiment_type(score)` - 分数映射：>0.7 正面，<0.3 负面，其余中性

**二、关键词提取**：
- `extract_keywords(content, topK)` - 使用 jieba **TF-IDF** 提取，支持停用词过滤、去重和词性过滤（名词、动词、人名、地名）
- `extract_keywords_bert(content, topK)` - 原始为 KeyBERT，降级为 TF-IDF 备用方案

**三、事件分类**：
- `categorize_event(title)` - 使用正则表达式匹配 8 个类别：社会民生、政治时事、财经金融、娱乐明星、科技互联网、体育赛事、文化旅游、健康养生，未匹配归类为"其他"

**四、文本预处理**：
- `preprocess_text(text)` - 删除 URL、@提及、话题标签、特殊字符，标准化空格
- `cut_text(text)` - jieba 分词
- `load_stopwords(filepath)` - 从文件加载停用词
- `append_sentiment_words()` - 追加自定义情感词

**设计特点**：内嵌约 100 个中文停用词，包括常见无意义热搜词（"男子"、"女子"、"网友"等）。

### 12.6 爬虫模块 (crawlers/)

#### `crawlers/weibo_crawler.py` - 微博主爬虫

**类**：`WeiboHotSearchCommentCrawler`

**技术栈**：Selenium + Microsoft Edge WebDriver + 用户配置文件免登录

**爬取流程**：
1. 启动 Edge 浏览器，加载用户配置文件
2. 访问微博热搜页面（`https://weibo.com/hot/search`）
3. 提取热搜榜单，获取前 N 个热搜话题

6. 通过 Kafka 发送评论/文章/事件数据到对应主题
7. 可选保存到本地 CSV 文件

**反爬策略**：随机延时操作，模拟人类浏览行为。

#### `spark_streaming/weibo_hot_consumer.py` - Spark 流消费者

**类**：`WeiboHotConsumer`

**定位**：PySpark Structured Streaming 应用，消费 Kafka 数据并写入 MySQL。

**数据流**：
```
Kafka (events/articles/comments 主题)
  -> PySpark Structured Streaming (JSON 解析)
    -> ETL 转换 (字段映射、清洗)
      -> MySQL upsert (hot_events/articles/comments 表)
```

**核心特性**：
- 消费 3 个 Kafka 主题（events、articles、comments）
- 使用 **Checkpoint** 实现 Exactly-Once 语义
- 配置 **背压**（`maxOffsetsPerTrigger=50`）控制消费速率
- 写入 MySQL 使用 upsert 操作（`ON DUPLICATE KEY UPDATE`）

**关键代码路径**：[spark_streaming/weibo_hot_consumer.py:1](spark_streaming/weibo_hot_consumer.py:1)

### 13.2 数据流向

```
微博热搜页面 ──(爬虫)──> Kafka ──(Spark Streaming)──> MySQL
                                    │
                                    └──(WebSocket)──> 前端实时展示

MySQL ──(数据模型)──> 业务服务 ──(Flask路由)──> HTML 页面
                        │
                        ├── 内存缓存 (CacheService)
                        └── Redis 缓存
```

### 13.3 关键技术特征

| 特征 | 实现方式 |
|------|----------|
| **NLP 情感分析** | SnowNLP + 自定义情感词典（300+ 词）校准 |
| **关键词提取** | jieba TF-IDF + 词性过滤 + 停用词过滤 |
| **事件分类** | 正则匹配 8 大类别 |
| **异常检测** | IsolationForest（100 棵树，0.05 污染率） |
| **影响力评估** | 五因子加权模型（热度、评论量、互动率、情感、时效） |
| **AI 报告** | MiniMax API（Anthropic 兼容端点） |
| **实时推送** | Kafka + WebSocket |
| **流处理** | PySpark Structured Streaming + Checkpoint |
| **缓存策略** | 内存缓存（5 分钟）+ Redis（分层 TTL） |
| **安全认证** | Flask Session + Werkzeug 密码哈希 + 密保问题 |
| **角色控制** | admin/user 角色区分，管理员可配置系统参数 |
| **数据导出** | CSV/ZIP 导出，支持日期范围和数据类型筛选 |
| **爬虫反检测** | Edge 用户配置免登录 + 随机延时 + 无头模式 |
